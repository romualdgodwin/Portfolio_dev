Projet Veliko

Objectif du projet

Développer une carte interactive répertoriant toutes les stations Vélib’ d’Île-de-France. Les utilisateurs peuvent consulter la disponibilité des vélos classiques et électriques, ajouter leurs stations préférées à une liste de favoris, et gérer leur compte via un espace client. Un espace administrateur permet la gestion centralisée des stations et des données.

Informations utiles

  • Langages utilisés: TWIG, PHP, JAVASCRIPT
  • Frameworks utilisés: Symfony
  • Base de données: MySQL
  • API : API Vélib’, OpenStreetMap

Défis rencontrés

Développement de l’interface

Il était essentiel de maîtriser la structure du framework TWIG et de faire des choix stratégiques pour les palettes de couleurs afin de garantir une expérience visuelle cohérente et attrayante. L’intégration d’animations modernes et la conception d’une interface intuitive ont permis de créer un site à la fois esthétique et fonctionnel, répondant aux standards actuels du design web.

Ajout de stations aux favoris

Cette fonctionnalité nécessitait une maîtrise approfondie de Symfony, ainsi que des ajustements dans la base de données pour intégrer les stations favoris. Il a été nécessaire de rédiger des requêtes SQL efficaces et de choisir les bonnes boucles pour récupérer les informations pertinentes à partir de l’API, assurant ainsi une gestion fluide et performante des données utilisateurs.

Affichage des favoris

L’ajout de stations aux favoris depuis la carte et à partir d’une liste dédiée sur une page séparée a présenté un véritable défi. La difficulté résidait dans l’affichage correct des stations, ce qui a nécessité plusieurs heures de travail. Après une analyse approfondie, il est apparu que le problème venait de la formulation de la requête SQL, que j’ai finalement optimisée pour résoudre le dysfonctionnement.
